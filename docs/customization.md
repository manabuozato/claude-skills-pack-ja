# カスタマイズガイド / Customization Guide

## スキルのカスタマイズ方法

各スキルの SKILL.md は、自分のワークフローに合わせて編集できます。

---

## 1. トリガーフレーズの変更

SKILL.md の冒頭にある「トリガー」セクションを編集します。

```markdown
**トリガー**: 「議事録」「minutes」「打ち合わせの内容整理して」
```

自分がよく使う言い回しを追加:

```markdown
**トリガー**: 「議事録」「minutes」「打ち合わせの内容整理して」「MTGまとめ」「会議ログ」
```

## 2. 出力フォーマットの変更

各スキルの「出力」セクションにあるテンプレートを編集します。

### 例: 日報にカテゴリを追加

```markdown
【本日の実績】

▼ 開発
- ...

▼ 営業
- ...

▼ 管理業務
- ...
```

カテゴリを自分の業務に合わせて変更してください。

### 例: 議事録にセクションを追加

```markdown
【リスク・懸念事項】
- [リスク事項]

【関連資料】
- [資料へのリンク]
```

## 3. 敬語レベルの変更

デフォルトは「だ・である調」ですが、SKILL.md 内の設定を変更できます。

```markdown
デフォルトは「です・ます調」とする。
```

## 4. MCP連携の設定

Notion、Google Calendar、Gmail との連携情報を SKILL.md に追記します。

```markdown
## MCP連携設定

- Notion TODOデータベース: [データベースURLまたは名前]
- 日報データベース: [データベースURLまたは名前]
- Googleカレンダー: プライマリカレンダーを使用
```

## 5. 言語設定

デフォルトは日本語出力ですが、以下を追記すると変更できます。

```markdown
## 言語設定
- 出力言語: 英語
- 日付フォーマット: YYYY-MM-DD
```

## 6. 独自スキルの作成

既存のスキルを参考に、自分だけのスキルを作ることもできます。

### スキルの基本構造

```markdown
# [スキル名]

## Description
[何をするスキルか、1-2行で]

**トリガー**: [起動キーワード]

## 処理手順
1. [ステップ1]
2. [ステップ2]
...

## 出力フォーマット
[テンプレート]

## カスタマイズポイント
[ユーザーが変更できる箇所]
```

この構造に沿って作成すれば、Claude.ai で動作するスキルになります。
