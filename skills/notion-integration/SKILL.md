# Notion連携スキル / Notion MCP Integration Patterns

## Description

Claude と Notion MCP を連携させて、TODO管理・日報更新・プロジェクト進捗追跡を行うためのパターン集です。

**トリガー**: 「Notion」「タスク追加」「タスク一覧」「日報をNotionに」「プロジェクト進捗」

**前提条件**: Notion MCPサーバーが有効であること

---

## パターン1: TODO管理

### タスク一覧の確認

```
「今日のタスクを見せて」
「優先度Aのタスクは？」
「期限が今週中のタスク一覧」
```

処理:
1. Notion のTODOデータベースを検索
2. フィルタ条件に応じて絞り込み
3. 優先度順で表示

### タスクの追加

```
「タスクを追加: [タスク名]、期限は[日付]、優先度A」
```

処理:
1. タスク名・期限・優先度をパース
2. Notion データベースに新規ページ作成
3. 作成完了を報告

### タスクの更新

```
「[タスク名]を完了にして」
「[タスク名]の期限を延長して」
```

処理:
1. 該当タスクを検索
2. ステータスまたはプロパティを更新
3. 更新完了を報告

---

## パターン2: 日報DB更新

### 日報の新規作成

```
「今日の日報をNotionに保存して」
```

処理:
1. 日報データベースを特定
2. 今日の会話履歴から日報内容を構成
3. 新規ページを作成（日付、実績、所感、明日の予定）
4. 作成されたページのURLを返す

### 日報の追記

```
「今日の日報に追記: [内容]」
```

処理:
1. 今日の日付で日報ページを検索
2. 既存のページに内容を追記
3. 更新完了を報告

---

## パターン3: プロジェクト進捗追跡

### 進捗の確認

```
「プロジェクトXの進捗を教えて」
「今週の進捗まとめ」
```

処理:
1. プロジェクトデータベースを検索
2. ステータス・完了率・残タスクを集計
3. サマリーを生成

### 週次レビュー

```
「今週の振り返りをまとめて」
```

処理:
1. 今週完了したタスクを取得
2. 進行中のタスクを取得
3. ブロッカーや懸念事項を抽出
4. 週次レポートを生成

## 推奨するNotionデータベース構造

### TODOデータベース

| プロパティ | 型 | 選択肢 |
|-----------|-----|--------|
| タスク名 | Title | - |
| ステータス | Select | 未着手 / 進行中 / 完了 / 待ち |
| 優先度 | Select | A / B / C |
| カテゴリ | Multi-select | 自由設定 |
| 期限 | Date | - |
| メモ | Rich text | - |

### 日報データベース

| プロパティ | 型 |
|-----------|-----|
| 日付 | Date |
| 実績 | Rich text |
| 所感 | Rich text |
| 明日の予定 | Rich text |

## カスタマイズポイント

- データベースのURLはプロジェクト設定で指定
- プロパティ名は自分のNotionに合わせて変更可能
- 「データベース一覧を見せて」でNotionのDB一覧を確認可能
