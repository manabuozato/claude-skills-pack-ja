# 週報生成スキル

## 固定ID

| 対象 | ID |
|------|-----|
| 日報ページ | 2d21bdf308098036befad550814ae0b6 |
| 日報DB | collection://2794b599-0e8b-4f4b-9652-51322dc71638 |
| TODOページ | 2d41bdf30809804cb599eab7e2c2b2c8 |
| ネタDB data_source | collection://9094ce3b-d70c-4c62-bc43-4c2eef701d03 |
| 戦略メモページ | 32d1bdf3-0809-81da-8e53-f67d4ce216c8 |
| Familyカレンダー | njl0td9u0cvo6sbo6p85c6pv5g@group.calendar.google.com |

## 文体ルール

ダッシュ系つなぎ禁止。太字強調禁止。新規追記分のみ適用。

## 実行手順

### Step 1: 対象週の日報を収集

1. 日報DB(collection://2794b599-0e8b-4f4b-9652-51322dc71638)から今週（月〜日）のページを取得
2. 日報DBに入っていない日報もある可能性があるため、日報ページ本文も確認
3. 全セクションを対象にする（月報と違い制限なし）

### Step 2: カテゴリ別まとめ

日報の各セクションからカテゴリ別に活動を集約:

| カテゴリ | 日報のセクション |
|---------|-----------------|
| UoC | UoC |
| 研究 | 研究・執筆 |
| CIRE | その他の活動内のCIRE関連 |
| その他 | その他の活動のCIRE以外 |

各カテゴリで主な活動を3〜5行程度に要約。

### Step 3: ミーティング集計

日報のミーティングセクションから今週のミーティングを集計。
件数と主要なもの（3〜5件）を記載。

### Step 4: 完了タスク

TODOページ(2d41bdf30809804cb599eab7e2c2b2c8)をfetch。
今週チェック済みのタスクを抽出（日報に記載されたものから判定）。

### Step 5: 来週の予定

Google Calendar primary + Family から来週（月〜日）の予定を取得。
主要なもの（ミーティング、締切、イベント等）を一覧化。

### Step 6: ネタDB状況

ネタDB(data_source: 9094ce3b-d70c-4c62-bc43-4c2eef701d03)から:
- 今週追加されたネタの件数と一覧
- 未整理の総件数
- ストックの総件数

### Step 7: 5資産進捗メモ

戦略メモページ(32d1bdf3-0809-81da-8e53-f67d4ce216c8)をfetch。
内容があれば5資産の進捗状況を引用。なければスキップ。

### Step 8: 週報生成

```
📋 第○週 週報（M/DD〜M/DD）

## 今週やったこと

### UoC
- 活動1
- 活動2

### 研究
- 活動1

### CIRE
- 活動1

### その他
- 活動1

## ミーティング
今週: ○件
- M/DD ○○さん（プロジェクト名）
- M/DD △△ミーティング
- ...

## 完了タスク
- タスク1
- タスク2
- ...

## 来週の予定
- M/DD ○○（時間）
- M/DD △△
- ...

## ネタDB
今週追加: ○件
未整理: ○件 / ストック: ○件

## 5資産メモ
（戦略メモから引用、またはスキップ）
```

### Step 9: 保存

1. チャットに週報を表示
2. 日報ページ内に「📋 第○週 週報（M/DD〜M/DD）」セクションとして保存
   - 月報セクションと同様、「✅過去の日報」セクションの直前に配置

## リマインド

朝コマンドのヘルスチェック（Step 6）で、金曜日に当週の週報が未作成の場合にリマインド表示される。

## 注意事項

- 日報が不足している場合は「○日分の日報が不足しています」と報告
- 5資産進捗は戦略メモがまだ未生成の場合はスキップ
- 週の区切りは月曜始まり
- 「先週の週報」と言われたら前週分を生成
