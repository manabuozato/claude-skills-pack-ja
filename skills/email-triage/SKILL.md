# メール仕分けスキル / Email Triage

## Description

複数のメールアカウントの未読メールをチェックし、緊急度・重要度で自動分類。返信ドラフトの生成も行います。

**トリガー**: 「メール確認」「メールチェック」「未読メール」「メール仕分け」「email triage」

**前提条件**: Gmail MCPサーバーが有効であること（複数アカウント対応）

---

## 処理手順

### 1. 未読メールの取得

Gmail MCPを使って、登録済みの全アカウントの未読メールを取得します。

```
各アカウントについて:
- 未読メールの件名、送信者、受信日時、本文冒頭を取得
- 件数を集計
```

### 2. 緊急度・重要度の判定

各メールを以下の4象限に分類します。

| | 緊急 | 非緊急 |
|---|------|--------|
| **重要** | 🔴 即対応 | 🟡 今日中 |
| **非重要** | 🟠 委任検討 | ⚪ 参考 |

**判定基準:**

🔴 即対応:
- 上司・クライアントからの直接依頼
- 「至急」「緊急」「ASAP」「urgent」を含む
- 締切が今日中

🟡 今日中:
- 返信が必要な依頼・質問
- 重要なプロジェクトに関する連絡
- 承認・確認の依頼

🟠 委任検討:
- CC で入っているが対応が必要そうなもの
- 自分以外でも対応可能な依頼

⚪ 参考:
- ニュースレター、通知
- CC のみ（情報共有目的）
- 自動通知メール

### 3. 出力フォーマット

```
━━━━━━━━━━━━━━━━━━━━━━━━
📧 メールトリアージ結果
━━━━━━━━━━━━━━━━━━━━━━━━

合計: [X]件の未読メール（[N]アカウント）

🔴 即対応（[N]件）
1. [送信者] — [件名]
   [本文の要約1行]
   → 推奨アクション: [返信 / 電話 / 対面]

🟡 今日中（[N]件）
1. [送信者] — [件名]
   [本文の要約1行]

🟠 委任検討（[N]件）
1. [送信者] — [件名]

⚪ 参考（[N]件）
- [件名] x [N]件

━━━━━━━━━━━━━━━━━━━━━━━━
```

### 4. 返信ドラフト生成（オプション）

ユーザーが「返信を書いて」と指示した場合:
- 該当メールの文脈を把握
- 適切な敬語レベルで返信ドラフトを生成
- 社内向け / 社外向けのトーンを自動判定

```
「1番のメールに返信して」
→ 該当メールの返信ドラフトを生成

「全部の返信を下書きして」
→ 🔴と🟡のメール全てにドラフトを生成
```

## カスタマイズポイント

- `「[アカウント名]だけ」` → 特定アカウントのみチェック
- `「今日のメールだけ」` → 日付フィルタ
- `「返信ドラフトも」` → 全件の返信ドラフトを自動生成
- `「英語メールは英語で返信」` → 言語を自動判定して返信
