# 月報生成スキル

## 固定ID

| 対象 | ID |
|------|-----|
| 日報ページ | 2d21bdf308098036befad550814ae0b6 |
| 日報DB | collection://2794b599-0e8b-4f4b-9652-51322dc71638 |
| Familyカレンダー | njl0td9u0cvo6sbo6p85c6pv5g@group.calendar.google.com |

## 文体ルール

ダッシュ系つなぎ禁止。太字強調禁止。新規追記分のみ適用。

## 月の判定ルール

- 「月報」のみ → 前月の月報を生成
- 「3月の月報」→ 指定月の月報を生成
- 月末5日前〜月末 → 当月の月報を生成（途中経過として）

## 実行手順

### Step 1: 対象月の日報を収集

1. 日報DB(collection://2794b599-0e8b-4f4b-9652-51322dc71638)から対象月のページを取得
2. 日報DBに入っていない日報もある可能性があるため、日報ページ(2d21bdf308098036befad550814ae0b6)の「✅過去の日報」セクション内の対象月トグルも確認
3. 両方から対象月の日報を網羅的に収集

### Step 2: 月報対象セクションの抽出

各日報から以下のセクションのみ抽出:
- ミーティング（カレンダー）
- UoC

以下は除外:
- その他の活動
- 研究・執筆
- プライベートの活動

「月報に採用」チェックが入っている日報を優先的に含める。

### Step 3: UoCフォーマットで月報を生成

```
# YYYY年M月 月報

## 期間
YYYY年M月1日〜M月末日

## ミーティング一覧

| 日付 | 相手・内容 | 備考 |
|------|-----------|------|
| M/DD | ○○さんとの打ち合わせ（プロジェクト名） | 要点 |
| ... | ... | ... |

## UoC関連活動

### プロジェクト別

#### プロジェクトA
- 活動内容1
- 活動内容2

#### プロジェクトB
- 活動内容1

## 主なアウトプット・成果
- 成果1
- 成果2

## 翌月の予定・課題
- 予定1
- 課題1
```

### Step 4: Notion保存

日報ページ内に「📊 YYYY年M月 月報」セクションとして保存。
「✅過去の日報」セクションの直前に配置。

### Step 5: docx出力

docxスキル(/mnt/skills/public/docx/SKILL.md)を読んでdocxファイルを生成。
ファイル名: `YYYY-MM_monthly-report.docx`
/mnt/user-data/outputs/ に出力。

### Step 6: チャット報告

```
📊 YYYY年M月 月報を作成しました
- ミーティング: ○件
- UoC活動: ○件
- Notionに保存済み
- docxファイルを出力済み
```

## 注意事項

- 日報DBにまだ入っていない日報がある場合は、ページ本文から直接抽出する
- ミーティング一覧は日付順にソート
- UoC活動はプロジェクト別にグルーピング（CFF、AI×マンガゼミ、研修、その他等）
- 「翌月の予定」はカレンダーから翌月の主要予定を取得して記載
- 月報の内容が薄い場合（日報が不足している場合）はManabuに「○日分の日報が不足しています」と報告
