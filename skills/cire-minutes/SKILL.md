# CIRE議事録作成スキル

CIREミーティングの文字起こしから構造化された議事録（3パート構成）を生成し、NotionミーティングDBに登録する。

## 文体ルール（全出力共通）
- ダッシュ系つなぎ（——、ーー、--等）禁止
- 太字強調（**）禁止。強調は見出し階層や箇条書き構造で表現

## 処理手順

### Step 0: モデル確認
- Opusで実行されている場合、ユーザーに通知する:「現在Opusで実行中です。CIRE議事録はSonnetで十分対応可能です。モデル切替を検討してください。」
- 通知後、ユーザーの指示に従う（続行 or 中断）

### Step 1: データ取得
- ファイル → `/mnt/user-data/uploads/` から読み込み
- Notion → Tactiq受信箱（3001bdf3080981dca3f6f08510bc8b23）をfetch → CIRE関連ページ特定

### Step 2: 前処理
全体通読→主要トピック・構造把握、音声認識誤変換の修正（英語モード誤認識に注意）、発言者特定（可能な範囲）、冒頭雑談・接続確認は除外。

### Step 3: 議事録生成（3パート構成）
下記「出力フォーマット」に従い議事録を生成する。

### Step 4: ミーティングDB登録
議事録生成後、NotionミーティングDBに登録する。

#### 4-1: 重複チェック
- ミーティングDB（data_source: `collection://77760381-fd3e-49e5-b5c3-d24418468b4f`）を検索
- 同日・同タイトル（またはCIREプロジェクトで同日）のエントリがないか確認
- 既存エントリがある場合（例: Zoom自動保存版）:
  - ユーザーに報告:「同日のCIREミーティングが既にDBにあります: [タイトル]（録音手段: Zoom）。上書き/統合/スキップのどれにしますか?」
  - ユーザー指示を待つ

#### 4-2: DB登録
以下のプロパティでページを作成する:

| プロパティ | 値 |
|---|---|
| ミーティング名 | 議事録から抽出したタイトル |
| date:日時:start | ミーティング日時（ISO-8601） |
| date:日時:is_datetime | 1 |
| プロジェクト | CIRE |
| ステータス | 完了 |
| 参加者 | 議事録の参加者リスト |
| 場所 | オンライン / 対面（場所名） |
| 録音手段 | 元データに応じて選択（Tactiq/Plaud/Zoom等） |
| 要約 | Claude生成の要約（3〜5文。主要議題、決定事項、次のアクションを簡潔にまとめる） |
| 議題 | 主要議題をカンマ区切り |
| アクションアイテム | パート2のToDoから主要項目を抽出 |

#### 4-3: ページ本文
ページの本文（content）は以下の構成:

```
## 議事録

（CIRE形式3パート議事録の全文をここに配置）

---

## 文字起こし（原文）

（前処理前の文字起こし全文をここに配置）
```

## 出力フォーマット

```markdown
# CIRE ミーティング議事録

- 日時: YYYY年MM月DD日（曜） HH:MM〜HH:MM（約XX分）
- 参加者: 氏名1、氏名2、…
- 場所: オンライン / 対面（場所名）

---

## パート1: ミーティング議事録

【1st Section】
### [大見出し]
#### [小見出し（必要に応じて）]
[本文...]

（内容に基づいて適切な数のSectionを作成）

---

## パート2: ToDo・スケジュール管理

### ToDo項目（優先度順）

※注意: 担当者名は会話の文脈から推測したものであり、正確でない可能性があります。必ず確認してください。

#### 優先度：高
- [ ] タスク名（担当: 氏名）（期限: YYYY/MM/DD）

#### 優先度：中/低
- [ ] タスク名（担当: 氏名）（期限: YYYY/MM/DD or 未定）

### 期限付きタスク（日付順）
#### YYYY/MM/DD
- 担当者: タスク内容

### 次回ミーティング
- 日時/参加者/主な議題/準備事項

### 今後の重要スケジュール
- YYYY/MM/DD: 内容

---

## パート3: キーワード・キーフレーズ抽出

### 基本キーワード（頻出する重要な単語や概念、10個）
### 会話忠実キーワード（話者が実際に使った言い回し、10個）
### 解釈キーワード（分析から導き出された概念や論点、10個）
### キーフレーズ（重要な論点を表す完全な文や句、10個）
```

## 執筆ルール
- 発言者の区別はしない。内容統合形式で記述
- 章立てはトピック別（時系列でなく）
- 専門用語・固有名詞は正確に維持
- 話し言葉の繰り返し・言い淀みは整理
- 文字起こしにない発言・エピソードを捏造しない
- 数値・固有名詞を勝手に変えない

## キーワード抽出の5方向
1. 重要なプロジェクト名や活動を指す用語
2. 課題や問題点を表す表現
3. 具体的な解決策やアプローチ
4. 組織の構造や機能に関する概念
5. 将来の展望や発展方向

## CIRE既知コンテキスト
- 組織: 一般社団法人 日本シェフ・イン・レジデンス応援協会
- 主要メンバー: 大里学（代表）、日高誠人、西木戸弓佳
- 主要プロジェクト: ジェイソン・マッキニー来日、米日財団助成、プレイブック、The Relay Awards、noteコンテンツ
- 関連KW: キッチンレジデンス、シェフネットワーク、五島列島、地域活性化、ビザ手続き

## 出力
- `/mnt/user-data/outputs/` にmarkdown保存
- ファイル名: `YYYY-MM-DD_CIRE議事録.md`
- present_filesで提示
- NotionミーティングDBに登録（Step 4）

## 要約生成ガイドライン
DB「要約」プロパティに入れる要約は以下の基準で生成する:
- 3〜5文の簡潔な日本語
- 構成: 主要議題 → 決定事項/合意事項 → 次のアクション/懸念事項
- 議事録パート1の内容を凝縮する（パート2のToDoは「アクションアイテム」プロパティで別途カバー）
- 例:「ジェイソン来日スケジュールの最終確認を行った。温泉津での滞在期間を4/15〜4/22に決定。ビザ申請は短期滞在で進める方針。次回までに受入先との日程調整を完了させる。」
