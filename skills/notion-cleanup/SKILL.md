# Notion巡回整理スキル

## 固定ID

| 対象 | ID |
|------|-----|
| Tactiq受信箱 | 3001bdf3080981dca3f6f08510bc8b23 |
| ミーティングDB data_source | collection://77760381-fd3e-49e5-b5c3-d24418468b4f |
| ネタDB data_source | collection://9094ce3b-d70c-4c62-bc43-4c2eef701d03 |
| ネタDBページ | 32d1bdf3-0809-811e-b24f-eaf9ccc9db80 |
| 運用リファレンス | 3001bdf308098182aac1dcc3b8e14859 |

## 文体ルール

ダッシュ系つなぎ禁止。太字強調禁止。新規追記分のみ適用。

## 実行手順

### Step 1: Tactiq受け箱の処理

1. Tactiq受信箱(3001bdf3080981dca3f6f08510bc8b23)をfetch
2. 子ページがゼロ → 「Tactiq受け箱: 空です」と報告してStep 2へ
3. 子ページがある場合:
   - 各子ページをミーティングDB(data_source: 77760381-fd3e-49e5-b5c3-d24418468b4f)へmove
   - プロパティ埋め（日時、プロジェクト、ステータス=完了、録音手段=Tactiq）
   - 要約生成（本文冒頭から要点を抽出）
   - 完了後「Tactiq受け箱: ○件をミーティングDBに登録しました」と報告

### Step 2: ミーティングDB要約生成

1. ミーティングDB(data_source: 77760381-fd3e-49e5-b5c3-d24418468b4f)を検索
2. 要約プロパティが空のレコードを取得
3. ゼロ件 → 「ミーティングDB要約: 全件生成済みです」と報告してStep 3へ
4. ある場合:
   - 各ページの本文（文字起こし）を読み、Claude要約を生成
   - 既存の要約（ソースAIサマリー等）がある場合はページ本文に「ソースサマリー」として残す
   - 件数が多い場合（5件超）はManabuに「○件あります。全部やりますか？」と確認
   - 完了後「ミーティングDB要約: ○件生成しました」と報告

### Step 3: ネタDB棚卸し

1. ネタDB(data_source: 9094ce3b-d70c-4c62-bc43-4c2eef701d03)でステータス=未整理のページを検索
2. ゼロ件 → 「ネタDB: 未整理なしです」と報告してStep 4へ
3. ある場合:
   - 件数と一覧をManabuに提示
   - 「ストックに昇格」「カテゴリ変更」「削除」等の指示を待つ
   - Manabuの指示に従ってプロパティを更新
   - 指示がなければ一覧表示のみで次へ

### Step 4: 重複・ゴミ検出

1. ミーティングDBで同日同時刻の重複レコードを検出
   - 重複がある場合: 優先度ルール（Zoom > Tactiq > Plaud）に基づき削除候補を提示
   - Manabuの確認後に削除
2. Infrastructure配下の不要ページ（空ページ、テスト用ページ等）があれば報告

### Step 5: レポート出力

チャットに最終レポートを表示:

```
📋 Notion巡回整理レポート
- Tactiq受け箱: ○件処理（または「空」）
- ミーティングDB: 要約○件生成（または「全件済み」）
- ネタDB: 未整理○件（一覧表示済み）
- 重複: ○件検出（または「なし」）
```

## 注意事項

- 削除系操作は必ずManabuに確認してから実行
- 要約生成で文字起こし全文を読むとトークンを大量消費するため、長い場合は冒頭3000文字程度で要約
- ネタDB棚卸しは一覧提示まで。判断はManabuに委ねる
