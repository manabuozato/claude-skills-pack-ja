# 日報アップデートスキル

## 固定ID

| 対象 | ID |
|------|-----|
| 日報ページ | 2d21bdf308098036befad550814ae0b6 |
| TODOページ | 2d41bdf30809804cb599eab7e2c2b2c8 |
| Tactiq受信箱 | 3001bdf3080981dca3f6f08510bc8b23 |
| Familyカレンダー | njl0td9u0cvo6sbo6p85c6pv5g@group.calendar.google.com |

## 文体ルール

ダッシュ系つなぎ禁止。太字強調禁止。新規追記分のみ適用。

## 実行手順（トークン節約型）

### Step 1: カレンダー取得（軽量）

Google Calendar primary + Family から対象日イベントを取得（Asia/Tokyo, 00:00-23:59）。
これはfetch不要でツール呼び出しのみなのでコスト低い。先に実行。

### Step 2: 日報ページfetch（注意: 軽量化必須）

日報ページをfetch。取得後、以下を区別する:
- 対象日の日報セクション → 編集対象。内容を保持
- 「✅過去の日報」セクション → 読み飛ばす。チェック済み日報の有無だけ確認
- 過去日報の本文は読まない、要約しない、コンテキストに保持しない

チェック済み過去日報があれば、日報DBに保存してからページから削除。

### Step 3: Tactiq受け箱 → ミーティングDB整理

#### Step 3a: ユーザーへの確認

日報作成を進める前に、まずユーザーに確認する:
「今日のミーティング文字起こしはTactiq受け箱に反映済みですか？」
- 「はい」「OK」等 → Step 3bへ進む
- 「まだ」「あとで」等 → Step 3b/3cをスキップし、Step 4へ進む（ミーティングDB内の既存データのみで日報作成）
- 「ない」「今日はミーティングなし」等 → 同上

#### Step 3b: 受け箱の確認と移動

Tactiq受信箱(3001bdf3080981dca3f6f08510bc8b23)をfetchし、子ページ一覧を確認。
ミーティングDBのデータソース: collection://77760381-fd3e-49e5-b5c3-d24418468b4f

- 子ページがゼロ → スキップ
- 子ページがある場合 → 全子ページをミーティングDBへmove（move-pages、parent=data_source_id: 77760381-fd3e-49e5-b5c3-d24418468b4f）

#### Step 3c: プロパティ埋め

移動した各ページのプロパティを埋める:
- ミーティング名: ページタイトルから判定
- 日時: タイトルの日付部分またはページ本文冒頭から抽出（is_datetime=1推奨）
- プロジェクト: タイトルやキーワードから判定（CFF/CIRE/研究/UoC/日置市/都政新報/博報堂/その他）
- ステータス: 完了
- 録音手段: Tactiq
- 参加者/要約/議題/場所: ページ本文から抽出可能な範囲で埋める（本文が長い場合は冒頭部分のみ確認）

プロパティ埋めは1件ずつfetch→update。件数が多い場合（10件超）は日時・プロジェクト・ステータス・録音手段の最低限のみ埋め、要約等は省略してよい。
完了後「Tactiq受け箱から○件をミーティングDBに整理しました」と報告。

### Step 4: ミーティングDB検索（日報用、一本化）

Notion searchでミーティングDB内の対象日レコードを検索。Tactiqから移動したものも、Plaud等で直接登録したものも、ここで一括取得。
- ヒットした場合のみ → 詳細fetchして日報に反映
- ない場合 → スキップ

### Step 5: TODO完了タスク振り分け

TODOページをfetch。対象日チェック済みタスクを特定し、日報セクションに振り分け。
判定方法:
- 見出し日付（M/DDやること）と対象日一致 → その日のタスク
- 「新規抽出」セクションのチェック済み → 対象日追加分
- 過去日報に記載済み → 除外

### Step 6: Slack書き込み確認

対象日のユーザー公開チャンネル書き込みを確認。日報に反映すべき活動を抽出。

### Step 7: 日報更新

既存手入力を保持しつつカレンダー情報をマージ。`(追記)` マーカーはミーティング内容で置換。

### Step 8: 翌日下書き（時刻判定）

- 対象日が当日 → 当日日報を再更新のみ
- 対象日が前日以前 → 過去に移動 + 当日下書き作成

## 日報フォーマット

```
## YYYY年M月D日（曜）

### ミーティング（カレンダー）
- イベント名（HH:MM-HH:MM）
  - 参加者: ○○、△△
  - 要点
  - [詳細](URL)  ← ある場合

### UoC
- 活動内容

### その他の活動
*月報には含みません*
- 活動内容

### 研究・執筆
*月報には含みません*
- 活動内容

### プライベートの活動
*月報には含みません*
- 活動内容
```

### カレンダー分類

| カレンダー | 分類先 |
|-----------|--------|
| primary（仕事） | ミーティング or UoC |
| Family | プライベートの活動 |
| Working Location | プライベート（「自宅」or「オフィス」） |

### 過去日報の整理

月別トグル（▶ 2026/02）配下に配置。各日報はトグル形式。「- [ ] 月報に採用」チェックボックス付与。

## 注意事項

- ユーザー手入力は絶対に消さない
- 同じイベントがprimary/Familyに重複 → 重複排除
- replace_content_range使用時は既存コンテンツの正確なマッチングに注意
