# 日報スキルの使用例

## 入力

ユーザーが「日報作って」と指示した後、以下の情報を提供:

```
今日やったこと:
- 10時からプロジェクトX定例。デザイン案Bに決定。
- 午後は提案書のドラフト作成。7割くらいまで進んだ。
- 16時からクライアントと電話。来週の打ち合わせ日程を調整。
- 夕方にバグ報告を3件対応。

明日は提案書を仕上げて、クライアントに送る予定。
最近タスクが増えてきて、優先順位の整理が必要だと感じた。
```

## 出力（生成された日報）

```
━━━━━━━━━━━━━━━━━━━━━━━━
日報 / Daily Report
━━━━━━━━━━━━━━━━━━━━━━━━

■ 日付: 2026年2月27日（木）
■ 氏名: （ユーザー名）

━━━━━━━━━━━━━━━━━━━━━━━━

【本日の実績】

▼ ミーティング
- [10:00] プロジェクトX定例 — デザイン案Bの採用を決定
- [16:00] クライアント電話 — 来週の打ち合わせ日程を調整

▼ タスク
- 提案書ドラフト作成 — 進捗70%
- バグ報告対応 — 3件完了

━━━━━━━━━━━━━━━━━━━━━━━━

【明日の予定】
- 提案書の完成・クライアントへ送付
- クライアント打ち合わせの準備

【所感・気づき】
タスク量が増加傾向にある。
優先順位の整理を行い、重要度の低いタスクの委任を検討したい。

【懸案事項】
- タスクの優先順位整理が必要

━━━━━━━━━━━━━━━━━━━━━━━━
```
